vscode：编辑器 xmake：基于 Lua 的轻量级跨平台构建工具 clangd：高性能，智能代码提示与补全 mingw：配置简单，生成的可执行文件体积小 cpptools：用于debug（可以使用codelldb代替）

安装环节#

安装好vscode以后，使用scoop安装xmake与clang,mingw

scoop install xmake
scoop install LLVM
#直接安装LLVM即带有clang，也可以单独安装，这里是为了方便
scoop install mingw

scoop会自动配置好环境变量，！！！重新启动终端！！！打开终端输入

xmake --version
clang -v
gcc -v
g++ -v

输出版本说明环境变量已经配置成功

接下来打开vscode安装插件 xmake clangd C++extensionpack（或者Codelldb）

vscode插件配置#

clangd#

打开vscode Clangd插件设置找到"clangd.arguments"

"clangd.arguments": [
  "--background-index",
  "-j=12",
  "--clang-tidy",
  "--all-scopes-completion",
  "--completion-style=detailed",
  "--header-insertion=iwyu",
  "--pch-storage=disk",
  "--compile-commands-dir=${workspaceFolder}/.vscode",
  "-query-driver=\"E:\\Apps\\MinGW\\mingw64\\bin\\g++.exe\"
  "
]

以上参数含义自行百度进行设置即可 –query-driver为电脑mingw的g++地址，不能直接复制

C/C++#

以下设置是为了避免C/C++插件的代码提示干扰clangd

打开vscode C/C++插件设置 搜索autocomplete设置为disable 搜索intelli，找到Intelli Sense Engine设置为disable

Codelldb#

如果你使用cpptools（C/C++ extension）调试可以跳过这部分

搜索xmake:debug设置Debug Config Type 为codelldb

实际运行#

运行#

mkdir Hello
cd Hello
code .

打开vscode以后新建hello.cpp

#include <iostream>

#include <map>


int main() {
    std::map<std::string, int> m;
    m["Alice"] = 30;
    m["Bob"] = 25;
    m.clear();
    std::cout<<"Hello World"<<std::endl;

}

然后在终端运行


xmake

当前版本（v2.9.5）会提示 note: xmake.lua not found, try generating it (pass -y or –confirm=y/n/d to skip confirm)?

输入y后vscode的xmake拓展自动激活

现在已经可以编译运行了（xmake会自动寻找到mingw，如果没有安装MSVC的话） 但是点击调试无法走到断点

调试#

接下来开始设置xmake.lua


target("hello")
    set_kind("binary")
    add_files("hello.cpp")
    set_toolchains("mingw")
    set_languages("c99", "cxx14")
    set_plat("mingw")
    add_rules("mode.debug", "mode.release")

在底部设置mode为debug，paltform修改Windows为mingw

点击小虫子即可调试，目前似乎无法绑定vscode的F5调试

ps：目前我拿来写算法，通常编译单一源文件，暂时没有测试交叉编译大型项目
